## 1. Executive Summary

- v0.0.7.x has produced a useful minimal scaffold sample and read-only check sample.
- The project should not continue expanding batch ID / profile / tag / metric content inside v0.0.7.x.
- Batch accidentTypeId / feedbackTag / triggerMetric / ingredient profile production is deferred to a later batch-content stage, likely v0.0.8.x or later. This is a direction note, not a confirmed release plan.
- The second half of v0.0.7.x should focus on closing the pre-tuning pipeline, preserving scaffold / check guardrails, and preparing a pre-shadow checklist.
- Codex must not generate game mechanism concepts by itself.

## 2. Why The Pivot Is Needed

The content dependency chain is coupled:

```text
accidentTypeId
-> triggerMetric
-> ingredient profile
-> threshold / severity
-> feedbackTag
-> review pack
-> shadow tuning
```

If v0.0.7.x starts batch-generating IDs before profiles, tables, review flow, validation, and shadow datasets are ready, the project risks:

- ID / profile mismatch
- duplicated concepts
- one-ingredient-one-accident drift
- docs / report bloat
- Codex-generated mechanism concepts
- future refactor before tuning even starts
- ID / profile / feedbackTag / severity layers drifting apart

The safer move is to close the route and defer bulk content production until the batch-content pipeline itself is ready.

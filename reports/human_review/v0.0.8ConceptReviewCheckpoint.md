## 4. Taste Line Summary

Source: `reports/human_review/approvedConceptList.taste.round1.draft.md`

当前 approved / revised concept list draft 包含：

1. 饮品存在感过低 / 太淡
2. 甜度过载
3. 酸度过载
4. 苦味过载
5. 咸味过载
6. 涩感 / 收敛感过强
7. 基础味觉整体过载

关键边界：

- 太淡不能简单写成 `if basicTasteSum < X then bland`；不能误伤牛奶、茶、咖啡等基础味觉低但风味身份 / body 清晰的饮品。
- 咸味过载来自 C005 revise；不要用咸甜失衡作为主概念，也不要惩罚合理咸甜组合。
- C007 特殊刺激不是单一 approved concept；它是 umbrella / splitNeeded / later review。
- C006 已单独保留涩感 / 收敛感。
- 辣感 / 麻感 / 酒精灼烧 / 挥发刺激等未来子概念需要单独 review。
- 基础味觉整体过载不能累加所有 flavor identity，不能误伤好喝的浓茶 / 花茶。
- 苦味 / 涩感 / 部分刺激通道需要未来 customer preference / tolerance。

## 5. Special Sensation Line Summary

Source: `reports/human_review/approvedConceptList.specialSensation.round1.draft.md`

当前 approved concept list draft 包含：

1. 辣感 / 灼辣感过强
2. 麻感 / 震麻感过强
3. 酒精灼烧 / 挥发刺激过强
4. 清凉刺激过强
5. 辛香刺激 / 香料压力

关键边界：

- Special sensation 子通道不能重新合并成一个粗暴的“特殊刺激过强”单机制。
- 顾客偏好 / tolerance 应接到具体 sensation channel，而不是统一 strongStimulation。
- 喜欢酒精灼烧不等于喜欢辣椒油；喜欢辣不等于喜欢麻、涩、清凉或辛香。
- 辛香刺激 / 香料压力是 cross-channel 概念：它有 flavor identity，但事故判断不应只靠 flavor identity。
- 辛香刺激 / 香料压力后续应通过 aromatic stimulation / spice pressure / intensity 等数值方向判断；这些只是方向词，不是本轮正式 triggerMetric。
- 不写单香料 if；不把姜、肉桂、八角、胡椒等具体香料直接写成机制身份。
- 特殊刺激主题饮 C006 保留为 high-level / overlap / later review，不作为普通机制，不急着结构化，不接 runtime。

## 6. Global Principles Already Confirmed

### Severity Identity Boundary

- 同一机制内通过 triggerMetric 区间、severityLevel、scoreMultiplier、feedback intensity 分轻中重。
- 不为了轻度 / 中度 / 重度拆 accidentTypeId。
- 水泥感 / 粉泥感只是示例，不是特例。
- 禁止用 `xxx_light`、`xxx_medium`、`xxx_heavy`、`xxx_severe` 之类 stable ID 表达强度。

### Cross-Layer Customer Preference / Tolerance

- Customer preference / tolerance 不是 taste-only。
- 未来 taste / texture / flavor / structure / severity / feedback / score 都可以被顾客偏好调节。
- 当前不创建 customerTag / audienceId。
- 当前不实现顾客系统。
- 未来应通过 preference profile / audience tags / tolerance / rule table / metadata 表达。
- 不写 `if 小孩 then 喜欢甜`、`if 上班族 then 喜欢苦咖啡` 之类硬编码。

### Single-Line Workflow / Parking Lot

- 默认一条设计线尽量闭环后再开启下一条。
- 新灵感、新债务和相邻方向先进入 Parking Lot。
- 只有明确阻塞当前线的问题，才临时处理，并在解除阻塞后回到原主线。

# Ingredient Schema

本文档记录原料数据的长期结构约定，方便后续增加原料、平衡数值或接入表格管理。

## 1. 当前数据位置

当前原料数据位于：

```text
data/ingredients.js
```

原料分类目前用于 UI 分组，不用于限制玩家选择。

玩家可以从任意分类自由添加多个原料。

## 2. 原料基础字段

建议每个原料长期保持这些信息：

```text
id: 稳定内部标识
name: 玩家可见名称
category: UI 分类
profile: 味觉和质地属性
tags: 规则标签
```

内部 id 应尽量稳定，避免破坏旧存档。

如果玩家可见名称变化，应优先保留旧 id，并做显示兼容。

例如：

```text
旧显示名：奶精
当前显示名：植脂奶
内部 id 可以暂时保持不变
```

## 3. 当前分类

当前 UI 分类包括：

- 茶类
- 乳类
- 液体
- 水果 / 风味
- 调味
- 小料

分类只影响展示位置，不限制配方结构。

## 4. 建议属性方向

长期可以围绕以下维度维护原料影响：

- 清爽度
- 厚重度
- 甜腻度
- 酸度
- 茶感
- 奶感
- 果感
- 气泡感
- 稠度
- 吸管阻力
- 猎奇度
- 成本
- 制作难度
- 拍照价值

其中：

```text
厚重度 / 油腻感
吸管阻力 / 固形物感
```

必须保持分离。

## 5. 原料标签建议

后续维护时可给原料增加标签，减少 if 判断堆积。

示例：

```text
tea
milk
highFatMilk
liquid
fruit
brightFruit
strongFlavor
solidTopping
muddyTexture
sweetener
industrialMilk
controversial
```

标签只服务规则判断，不应限制玩家选择。

## 6. 高风险原料

以下原料后续调整时要特别谨慎：

- 柠檬：少量提亮，过量酸度事故
- 榴莲：香气冲击、厚重、黏腻、争议、一定吸管阻力
- 芋泥：厚重、泥状、吸管阻力
- 奥利奥碎：固形物、甜腻、吸管阻力
- 淡奶油：高脂、高油腻，不是固体
- 厚乳：高奶感、高厚重，不是固体
- 奶盖：高奶感、高厚重，不应单独触发水泥感
- 植脂奶：低成本、工业感、廉价感、健康疑虑

## 7. 旧存档兼容

旧存档兼容优先级高于命名洁癖。

如果旧存档里存在旧名称或旧 id，应在展示和载入时兼容，不要让玩家保存的配方坏掉。

重点兼容：

```text
奶精 -> 植脂奶
```

## 8. 后续表格管理方向

未来如果原料越来越多，可以把原料数据迁移为可维护表格，再导出为运行时 JS 数据。

但当前阶段不要引入数据库、后端或复杂构建流程。

# Test Cases

## 1. 当前 v0.0.5.x 回归测试原则

v0.0.5.x 重点是味觉系统数据化地基。除非任务明确说明要调味觉，否则测试重点仍是确认没有大面积输出回退：正常配方不要变灾难，事故配方不要被洗白，UI 和交互不要受影响。

## 2. 自动化验收优先原则

在每次版本修改后，Codex 应优先尝试使用可用的浏览器自动化方式进行验收，例如 Codex 内置 Browser、Playwright、Puppeteer 或项目已有测试工具。

优先自动检查：

- 页面是否能正常打开
- 控制台是否有红色 error
- 顶部版本号是否正确
- 原料按钮加入 / 移除是否正常
- 已选状态是否同步
- 比例条是否保持总量不超过 100
- 手动调整一个原料时其他原料是否不被偷偷联动
- 试喝报告是否正常显示
- 是否出现 `undefined`
- 是否出现 `[object Object]`
- 保存 / 载入 / 删除测试配方是否正常

用户主要负责抽样试玩和产品判断，不应长期充当人肉 QA。

如果自动化验收不可用，Codex 必须说明失败原因，并给出用户需要手动检查的最小 P0 清单。

## 3. 测试分级原则

### P0：每次必测

- 页面能打开
- 控制台无红色 error
- 能添加一个原料
- 能试喝一次
- 核心按钮不报错

### P1：相关模块变更时测试

- 原料按钮开关逻辑
- 比例滑条
- 保存 / 载入
- 随机乱摇
- 代表性配方回归样本

### P2：大版本发布前测试

- 完整味觉回归
- 隐藏配方
- 经营长流程
- 多轮试玩手感

## 4. 必测项目

### 页面打开

- 页面能正常打开
- 顶部版本号显示正常
- Candidate 冻结前，页面顶部版本号必须与准备创建的 candidate tag 版本一致。
- 如果页面版本号落后，即使功能和测试都通过，也不能打 candidate tag。
- v0.0.5.6-candidate 存在页面显示仍为 v0.0.5.5 的已知小瑕疵；后续版本从 v0.0.5.7 起严格执行该检查。
- 控制台无 error

### 原料按钮

- 点击未加入原料，应加入当前杯子
- 再次点击已加入原料，应从当前杯子移除
- 不应出现每点一次 +10% 的累加逻辑
- 已选状态高亮 / ✓ 正常
- 删除、清空、随机乱摇、载入保存配方后，左侧按钮状态同步正常

### 比例条

- 手动改一个原料时，其他原料不动
- 总量可以低于 100%
- 总量不能超过 100%
- 总量为 100% 时，新加入原料应以 0% 或安全方式加入，不应突破 100%

### 核心功能

- 试喝一下可正常运行
- 随机乱摇可正常运行
- 清空杯子可正常运行
- 保存配方可正常运行
- 查看已保存配方可正常运行

## 5. 当前基础测试配方

### 清爽水果茶

绿茶 40 + 气泡水 30 + 柠檬 20 + 蜂蜜 10

预期：清爽水果茶，高分。

### 经典奶茶

红茶 45 + 牛奶 40 + 珍珠 15

预期：经典奶茶，中高分，反馈稳定，不要说清爽。

### 高级厚乳款

乌龙茶 40 + 厚乳 35 + 奶盖 20 + 海盐 5

预期：高级厚乳款，中高分，反馈体现高级感。

### 极端柠檬事故

柠檬 85 + 绿茶 5 + 气泡水 5 + 蜂蜜 5

预期：酸度事故，低分，反馈直接毒舌。

### 极端榴莲事故

榴莲 80 + 牛奶 10 + 红茶 10

预期：香气冲击 + 厚重 + 黏腻 + 一定吸管阻力。

### 奶脂过载

厚乳 70 + 淡奶油 20 + 植脂奶 10

预期：奶脂过载 / 高油腻事故。

### 吸管阻力事故

芋泥 45 + 奥利奥碎 32 + 珍珠 16 + 蜂蜜 7

预期：吸管阻力事故，低分，反馈说吸不上来 / 人生阻力 / 需要勺子。

### 气泡奶油冲突

气泡水 50 + 淡奶油 40 + 蜂蜜 10

预期：气泡和奶油冲突，不要说水泥。

### 工业奶茶

植脂奶 70 + 红茶 20 + 黑糖 10

预期：工业奶茶 / 廉价感 / 健康疑虑方向。

## 6. v0.0.5.0 数据化地基回归样本

### 正常组合

- 红茶 + 牛奶：应偏经典奶茶方向，不应被判成灾难。
- 绿茶 + 柠檬 + 蜂蜜：应偏清爽茶饮 / 水果茶方向，不应被判成灾难。
- 乌龙茶 + 厚乳：应偏顺口 / 高级奶茶方向，不应被判成灾难。
- 咖啡 + 牛奶：应偏咖啡奶饮方向，不应被判成灾难。

### 冲突组合

- 气泡水 + 淡奶油：应有质地冲突或奇怪感。
- 柠檬 + 牛奶：应有明显不协调或事故感。
- 榴莲 + 咖啡：应有香气冲突或压迫感。
- 芋泥 + 气泡水：应有泥浆感 / 质地奇怪感。

### 极端比例

- 榴莲极高比例：不应只说香气强，也应体现厚重、黏腻、压迫感。
- 淡奶油 / 厚乳 / 奶盖极高比例：不应温和夸成甜品感，应体现油腻、负担、奶脂过载方向。

## 7. 未来味觉系统测试方向

v0.0.5.x 之后逐步加入：

- 基础经典饮品测试
- 极端事故测试
- 边界比例测试
- 多水果茶 / 花果茶泛化测试
- 同材料不同液体支撑比例测试，用于观察 `drinkability` 和 `strawResistance` 是否能区分正常版与半固体翻车版。
- 温度 / 冰量 / 糖度测试
- 客群偏好测试
- 随机压力测试
- 金标样本回归测试

## 8. 结构事故规则表测试方向

后续治理结构事故规则时，应重点确认：

- 经典奶茶、正常厚重饮品不应被误判为结构事故。
- 高固体负载 + 低液体支撑应触发或保持口感事故方向。
- 同材料不同液体支撑比例应逐步成为回归样本，例如正常芋泥茶饮与半固体芋泥事故版本。
- 现有吸管阻力事故不能被结构规则洗白。
- 如果旧吸管阻力事故已命中，结构规则不应重复叠加过重惩罚。

## 9. 已知限制

- 当前保存配方功能使用浏览器本地存储。
- 旧保存配方可能因打开地址、端口、域名、`file://` 与 `localhost` 切换、浏览器清理数据等原因不可见。
- 这是原型期已知限制，不在 v0.0.4.x 修复。
- 后续可考虑导出 / 导入 JSON、存档版本号和正式本地存档方案。

## 10. 比例段规则表测试方向

后续治理比例段规则时，应重点确认：

- 迁移规则前后，代表样本输出应尽量保持稳定。
- 强风味原料比例边界需要回归样本。
- 未来可建立同一原料低 / 中 / 高 / 极端比例金标样本。
- 特别关注柠檬、榴莲、芋泥、奥利奥、乳脂类和小料类。
- 柠檬、榴莲等已迁移规则应保持原有方向，不应让事故样本被洗白，也不应误伤经典奶茶和普通饮品。

## 11. 金标样本 / 回归样本

金标样本用于防止重构回退。它不是隐藏配方识别，不是图鉴系统，也不是玩家可见的新玩法。

当前样本文件：

```text
data/goldenSamples.js
```

当前检查脚本：

```text
node scripts/runGoldenSamples.js
```

样本预期使用类型关键词、禁用类型、分数区间和反馈关键词，不使用反馈全文硬匹配。这样可以保留反馈文案随机选择的空间，同时检查核心判断方向是否稳定。

后续拆 `accidentAnalyzer.js`、`feedbackEngine.js`、`drinkTypeAnalyzer.js` 或继续迁移比例 / 事故规则前，应先运行该脚本。重点确认：

- 正常配方不要被误伤成事故。
- 事故配方不要被洗白。
- 强风味边界样本保持稳定。
- 结构对照样本能区分“厚重但可饮用”和“高固体低液体”。

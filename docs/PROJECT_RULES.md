# 奶茶实验室项目长期规则

1. 《奶茶实验室》不是手忙脚乱接单摇奶茶的游戏，而是饮品研发、配方实验、试喝吐槽、经营验证的游戏。
2. 当前阶段是配方实验室测试版，暂时不做营业系统、员工系统、隐藏配方系统。
3. 配方系统是成分池 + 比例系统，不是每类只能选一个。
4. 茶底和乳基分开，“奶茶”不是茶底，而是茶 + 乳基的结果。
5. 味觉审判优先级固定为：极端比例事故 > 质地事故（奶脂过载/吸管阻力） > 冲突组合 > 正常好组合 > 普通分类。
6. 隐藏配方以后走身份识别线，不是免死金牌；极端事故仍然应该优先被识别。
7. 测试期可以显示完整属性条；正式版默认不展示所有后台数值。
8. 宽屏桌面优先，竖屏只是兼容。
9. 版本号使用四段：`vA.B.C.D`。
   - A = 测试/正式阶段
   - B = 大功能阶段
   - C = 重大逻辑/结构变化
   - D = 小修补

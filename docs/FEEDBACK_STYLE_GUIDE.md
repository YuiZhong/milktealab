# Feedback Style Guide

本文档记录《奶茶实验室》试喝反馈的文案风格，防止后续变成普通评分器或客服腔。

## 1. 核心风格

试喝反馈是项目灵魂。

反馈应该：

- 毒舌但可爱
- 有画面感
- 能让玩家意识到系统读懂了配方
- 正常组合夸得具体
- 离谱组合吐槽得有记忆点
- 事故组合好笑但不打击玩家继续实验

不要写成：

- 客服话术
- 正式质检报告
- 只有“不错 / 一般 / 较差”的空话
- 没有配方指向性的泛泛评价

## 2. 正常组合反馈

正常组合要明确告诉玩家“为什么成立”。

方向：

- 经典
- 稳定
- 顺口
- 清爽
- 层次好
- 有明确客群

示例方向：

```text
红茶和牛奶配得很稳，是不会出错的经典款。
这杯不花哨，但很会做人。
乌龙和厚乳配得很稳，奶盖把层次托起来了。
这杯有点高级，像突然知道自己要卖给谁的饮料。
```

## 3. 花果茶反馈

花果茶不能轻易写成实验特调。

方向：

- 茶香托住水果
- 果味清楚
- 蜂蜜或少量水让整体更顺
- 茉莉茶适合清香水果
- 绿茶适合清爽水果
- 乌龙茶适合成熟水果感

示例方向：

```text
花香和果香接上了，像一杯认真上班的水果茶。
桃子和葡萄没有抢戏，茶底把它们稳稳托住了。
茉莉茶在这里很会做人，果香变得轻而不飘。
```

## 4. 极端事故反馈

极端比例事故可以更直接、更戏剧化。

玩家已经知道自己在乱来，反馈可以吐槽得更狠，但不要恶意攻击玩家本人。

示例方向：

```text
这不是清爽，是柠檬在杯子里发动总攻。
酸度已经不是提神，是在和味蕾打架。
试喝员喝完没说话，只是把工牌摘下来了。
这杯不是饮料，是写给味蕾的恐吓信。
这不是试喝，是一次工伤。
```

## 5. 奶脂过载反馈

奶脂过载不是吸管阻力。

它的重点是：

- 腻
- 重
- 负担
- 喝得动但喝不下去
- 第一口快乐，后面难受

不要写：

- 吸管辞职
- 水泥
- 半固体
- 需要装修队

示例方向：

```text
第一口像奖励，第二口像惩罚，第三口开始怀疑人生。
奶味不是越多越好，这杯已经把幸福熬成了负担。
它不是吸不上来，是喝下去以后不太下得去。
奶脂感已经不是香，是压迫。
```

## 6. 吸管阻力反馈

吸管阻力重点是物理层面的难吸。

方向：

- 吸不上来
- 需要勺子
- 半固体
- 像水泥 / 装修材料
- 更像甜品糊，不像饮料

示例方向：

```text
吸管刚插进去就提交了辞职信。
这杯不是饮料，是需要装修队施工的半固体。
试喝员努力吸了一口，结果只吸到了人生的阻力。
它不是难喝，它是物理意义上很难喝到。
```

## 7. 植脂奶反馈

植脂奶不是普通牛奶，也不是厚乳。

方向：

- 便宜
- 工业感
- 廉价感
- 配料表压力
- 学生党可能接受
- 白领、健身党、老人可能不喜欢

示例方向：

```text
奶味是有了，但像是从配料表里硬挤出来的。
这杯喝起来很努力地想装成牛奶。
试喝员看了一眼配料，开始担心自己的体检报告。
它不是不好喝，是有一种很熟悉的工业快乐。
```

## 8. 冲突组合反馈

冲突组合要说明冲突来源，不要只骂。

方向：

- 口感分裂
- 香气打架
- 酸奶化 / 结块联想
- 泥浆快乐水
- 试喝员思考人生

示例方向：

```text
气泡和奶油在杯子里打架，试喝员正在思考人生。
榴莲和咖啡都很有主见，问题是它们谁也不听谁的。
这杯很有想法，问题是想法太多，已经开会吵起来了。
```

## 9. 禁止方向

不要让反馈变成：

- “本产品口感较好，适合年轻人”
- “此饮品酸甜适中，建议优化比例”
- “味道一般，请继续努力”
- “根据计算，该配方存在问题”

玩家要感觉自己在和一个有性格的试喝员互动，而不是在看后台报表。
